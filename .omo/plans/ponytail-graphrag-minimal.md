# Ponytail GraphRAG Minimal Flow Plan

## TL;DR
> Summary:      Verify Ponytail is globally installed for Codex, then make the existing t08_project GraphRAG analysis flow more selective, traceable, private, and honest with minimal code. Keep the current SQL-first schema, avoid Neo4j/Microsoft GraphRAG/embeddings, and replace fake frontend score percentages with source-backed priorities.
> Deliverables:
> - Ponytail global Codex install and hook/Node verification receipts.
> - Ranked top-k public KG facts instead of all company facts.
> - Taxonomy-backed study/interview context from existing role/skill tables.
> - Normalized backend roadmap response with source traceability.
> - Frontend result UI without fabricated deterministic scores.
> - API/UI/private-data isolation evidence and cleanup receipts.
> Effort:       Medium
> Risk:         Medium - cross-layer contract changes touch backend KG retrieval, LLM payload/prompt, persisted analysis JSON, and result-page UI.
