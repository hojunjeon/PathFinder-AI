# PathFinder AI — Agent Guide

## Architecture (3-tier monorepo)

| Directory | Tech | Port | Entrypoint |
|-----------|------|------|------------|
| `frontend/` | Vue 3 + Vite + Pinia + Vue Router | 5173 | `src/main.js` → `src/router/index.js` |
| `backend/` | Django 5.2 + DRF + SimpleJWT | 8080 | `config/settings.py` → `config/urls.py` |
| `llm_server/` | FastAPI + httpx | 8081 | `main.py` |

Custom `AUTH_USER_MODEL = 'accounts.User'`. All backend API routes live under `/api/`.

LLM server is a thin proxy: `/llm/roadmap` receives structured profile/job data, builds a Korean prompt, and calls OpenAI (via SSAFY GMS gateway) or falls back to mock responses.

## Dev commands

```bash
# --- Start all 3 servers (Windows only) ---
.\run-dev.bat
# Or set GMS_KEY first for real AI responses:
set GMS_KEY="your_key" && .\run-dev.bat

# --- Backend tests (pytest-django) ---
cd backend && python -m pytest

# --- LLM server tests (pytest) ---
cd llm_server && python -m pytest

# --- Frontend E2E (Playwright) ---
cd frontend && npx playwright test
# Requires all 3 servers running (GMS_KEY optional)
```

## Key gotchas

- **LLM server requires `X-Internal-Token` header** on every request. If `LLM_INTERNAL_TOKEN` env var is unset, the dev script auto-generates one and logs it. Without it, all endpoints return 503.
- **`GMS_KEY` required for real AI** — without it, `/llm/roadmap` returns hardcoded mock data marked `(Mock)`.
- **Frontend JWT tokens** stored in `localStorage` under `access`/`refresh`. Axios interceptor at `src/api/index.js` auto-refreshes on 401.
- **Python 3.11 required** for the dev script's Python resolution.
- **Django runs with `--noreload`** in dev script (separate server processes).

## Testing quirks

- Backend tests follow `tests/test_*.py` pattern in each Django app (no top-level `tests.py`). See `backend/tests/test_test_discovery.py` for the enforced convention.
- LLM server tests use `monkeypatch` for env vars and mock the `httpx.AsyncClient`.
- E2E tests are in `frontend/tests/e2e/` — Playwright launches Vite automatically (`reuseExistingServer` unless CI).

## Environment variables

| Variable | Purpose | Default |
|----------|---------|---------|
| `GMS_KEY` | SSAFY GMS API key for LLM calls | (none, mock fallback) |
| `LLM_INTERNAL_TOKEN` | Auth token between backend and LLM server | auto-generated by dev script |
| `LLM_MAX_PROMPT_CHARS` | Max prompt character length | `12000` |
| `LLM_MAX_REQUEST_BYTES` | Max request body size | `2621440` (2.5 MB) |
| `LLM_ALLOWED_CLIENT_HOSTS` | IP whitelist for LLM server | `127.0.0.1,::1,testclient` |
| `DJANGO_SECRET_KEY` | Django secret (dev fallback when DEBUG) | hardcoded dev key |
| `DJANGO_DEBUG` | Django debug mode | `true` |
| `LLM_SERVER_URL` | Backend → LLM server proxy URL | `http://127.0.0.1:8081` |
| `DJANGO_CORS_ALLOWED_ORIGINS` | CORS origins | `localhost:5173, 127.0.0.1:5173` |

## LLM model

The FastAPI server calls `gpt-5-nano` via SSAFY GMS gateway at `https://gms.ssafy.io/gmsapi/api.openai.com/v1/chat/completions`. Prompt is in Korean. Response is JSON parsed from the LLM output with a regex fallback.
