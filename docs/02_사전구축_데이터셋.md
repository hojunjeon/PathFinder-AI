# PathFinder AI 사전 구축 데이터셋

## 목적

이 문서는 사용자가 이력서, 자기소개서, 채용공고를 입력하기 전에 데이터베이스에 미리 구축되어 있어야 하는 데이터를 정의하고, 이번에 실제로 구축한 데이터셋의 범위와 적재 방법을 정리한다.

이번 구축은 기존 Django 모델 형태를 유지하는 것을 전제로 했다. 따라서 새 테이블을 만들지 않고 현재 `Company`, `Job` 모델에 맞춰 데이터를 구성했다.

## 사용자 입력 전 사전 구축 데이터

사용자 입력 전에 DB에 있어야 하는 데이터는 서비스가 분석 기준으로 삼는 기준 데이터다.

| 구분 | 현재 저장 위치 | 설명 |
|---|---|---|
| 회사명 | `Company.company_name` | 대기업 이름 |
| 산업 | `Company.industry` | 회사가 속한 산업군 |
| 기업 규모 | `Company.size` | 이번 데이터셋은 모두 `large` |
| 인재상 | `Company.talent_description` | 기업 분석과 컬처핏 분석에 사용할 요약 문장 |
| 조직문화 키워드 | `Company.culture_keywords` | 협업, 품질, 혁신, 안전 등 |
| 직무명 | `Job.job_title` | 엔지니어링 직무명 |
| 연봉 추정값 | `Job.annual_salary_krw` | 대시보드/시장 분석용 수치 |
| 요구 경력 | `Job.required_experience_years` | 신입~주니어 범위 분석용 |
| 예상 지원자 수 | `Job.applicant_count` | 경쟁률/시장 분석용 수치 |
| 면접 단계 | `Job.interview_stages` | 코딩테스트, 기술면접, 실무면접, 인성면접 등 |
| 요구 역량 | `Job.required_skills` | 직무 분석과 역량 gap 분석 기준 |
| 직무 설명 | `Job.job_description` | 회사/산업 맥락에서 직무가 수행하는 역할 |
| 우대사항 | `Job.preferred_qualifications` | 자소서/이력서와 매칭할 보조 기준 |
| 추천 학습 영역 | `Job.recommended_study_areas` | 로드맵과 예상 질문 생성의 핵심 기준 |

## 사용자 입력 후 저장되는 데이터

사용자가 서비스를 이용하면서 입력하거나 생성되는 데이터는 사전 구축 데이터와 분리된다.

| 구분 | 현재 저장 위치 | 설명 |
|---|---|---|
| 사용자 계정 | `User` | 이메일 기반 인증 정보 |
| 프로필 | `Profile` | 전공, 학력, 경력, 프로젝트, 자격증, 수상내역 |
| 자기소개서 | `Profile.cover_letters`, `Analysis.submitted_cover_letter` | 저장된 자소서 또는 분석 시 제출한 자소서 |
| 채용공고 URL | `Analysis.job_posting_url` | 사용자가 입력한 공고 URL |
| 채용공고 본문 | `Analysis.job_posting_text` | 직접 입력하거나 파싱된 공고 본문 |
| 선택 면접 유형 | `Analysis.selected_interview_types` | 기술면접, 코딩테스트, PT, 인성 등 |
| 분석 결과 | `Analysis.competency_gap`, `Analysis.text_roadmap`, `Analysis.timeline_data` | LLM과 기준 DB를 조합해 생성한 결과 |

## 데이터 생성 방식

데이터는 현재 프로젝트의 모델에 바로 적재할 수 있도록 JSONL 형식으로 생성했다.

각 레코드는 아래 구조를 가진다.

```json
{
  "company_name": "삼성전자",
  "industry": "반도체/전자",
  "size": "large",
  "talent_description": "창의와 도전으로 기술 혁신을 주도하는 인재",
  "culture_keywords": ["글로벌", "혁신", "품질", "도전"],
  "job_title": "신입 백엔드 엔지니어 트랙 00001",
  "annual_salary_krw": 52000000,
  "required_experience_years": 0,
  "applicant_count": 80,
  "interview_stages": [
    {"order": 1, "type": "coding_test", "desc": "코딩테스트 또는 과제 전형"},
    {"order": 2, "type": "technical", "desc": "직무/기술 면접"},
    {"order": 3, "type": "personality", "desc": "인성 및 조직 적합성 면접"}
  ],
  "required_skills": ["API 설계", "데이터베이스", "트랜잭션", "분산 시스템", "장애 대응"],
  "job_description": "회사/산업 맥락 기반 직무 설명",
  "preferred_qualifications": ["관련 전공 또는 동등한 프로젝트 경험"],
  "recommended_study_areas": ["HTTP", "REST API", "DB 인덱스", "트랜잭션 격리수준", "캐시 전략"]
}
```

## 적재 방법

개발 환경에서 Django 의존성이 설치된 상태라면 아래 명령으로 DB에 적재할 수 있다.

```bash
cd backend
python manage.py seed_engineering_jobs
```

데이터 파일을 재생성하려면 프로젝트 루트에서 아래 명령을 실행한다.

```bash
python3 scripts/generate_large_company_engineering_dataset.py
```

## 서비스 활용 방식

분석 생성 시 현재 구조에서는 `Analysis.job`이 특정 `Job` 레코드를 참조한다. 따라서 사용자가 채용공고 URL 또는 본문을 입력하면 다음 방식으로 활용할 수 있다.

1. URL 또는 회사명으로 `Company` 후보를 찾는다.
2. 사용자가 `Job` 후보 중 하나를 선택한다.
3. 선택된 `Job.required_skills`와 `Job.recommended_study_areas`를 LLM payload에 넣는다.
4. 사용자의 프로필, 자기소개서, 채용공고 본문과 비교한다.
5. 부족한 직무 지식, 우선 학습 개념, 예상 질문을 생성한다.

## 한계와 추후 개선

이번 데이터셋은 현재 스키마를 유지하기 위한 기준 데이터셋이다. 실제 운영 품질을 더 높이려면 다음 개선이 필요하다.

- 실제 채용공고 원문과 출처 URL 저장
- 회사/직무/산업 데이터의 출처와 검수 상태 관리
- `Skill`, `KnowledgeConcept`, `InterviewQuestion` 별도 테이블 분리
- 회사별 실제 면접 후기 또는 공개 채용 프로세스 반영
- 연봉, 지원자 수 등 수치 데이터의 실제 출처 기반 보정
- 데이터 최신화 주기와 관리자 검수 프로세스 추가

현재 단계에서는 기존 모델을 유지하면서도 서비스 핵심 흐름인 기업/산업/직무 기반 면접 준비 추천을 구현할 수 있는 최소 기준 데이터가 구축된 상태다.
