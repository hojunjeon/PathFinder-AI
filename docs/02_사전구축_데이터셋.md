# PathFinder AI 사전 구축 데이터셋

## 목적

이 문서는 `backend/companies/data/large_company_engineering_jobs.jsonl` 파일의 현재 계약을 정리한다.
이 파일은 더 이상 합성 엔지니어링 직무 10,000건을 담지 않는다. 실제 웹 출처를 확인한 **대기업 회사 메타데이터**만 저장하며, Django의 `Company` 테이블만 갱신한다.

## 데이터 계약

각 JSONL 레코드는 아래 5개 필드만 가진다.

| 필드 | 저장 위치 | 설명 |
|---|---|---|
| `company_name` | `Company.company_name` | 회사명 |
| `industry` | `Company.industry` | 회사가 속한 산업군 |
| `size` | `Company.size` | 현재는 모두 `large` |
| `talent_description` | `Company.talent_description` | 기업 인재상/핵심가치/일하는 방식 기반 요약 |
| `culture_keywords` | `Company.culture_keywords` | 조직문화·핵심가치 키워드 배열 |

## 구축 원칙

- 데이터는 실제 웹 출처를 확인한 회사 메타데이터만 사용한다.
- 허용 필드는 `company_name`, `industry`, `size`, `talent_description`, `culture_keywords` 뿐이다.
- `job_title`, `annual_salary_krw`, `required_skills` 같은 직무 합성 필드는 이 파일에 넣지 않는다.
- 이 파일은 `Company` 데이터 적재용이며 `Job` 테이블을 생성하거나 갱신하지 않는다.

## 레코드 예시

```json
{
  "company_name": "삼성전자",
  "industry": "반도체/전자",
  "size": "large",
  "talent_description": "‘기업은 사람이다’라는 신념을 바탕으로 인재를 소중히 여기고 마음껏 능력을 발휘할 수 있는 기회의 장을 만들어 갑니다.",
  "culture_keywords": ["인재제일", "최고지향", "변화선도", "정도경영", "상생추구"]
}
```

## 적재 방법

개발 환경에서 Django 의존성이 설치된 상태라면 아래 명령으로 DB에 적재할 수 있다.

```bash
cd backend
python manage.py seed_engineering_jobs
```

이 커맨드는 다음만 수행한다.

1. JSONL 각 행의 필수 필드 5개를 검증한다.
2. `size` 가 `large` 인지 확인한다.
3. `Company` 를 `update_or_create` 한다.
4. `Job` 테이블은 건드리지 않는다.

## 현재 범위

- 저장 파일: `backend/companies/data/large_company_engineering_jobs.jsonl`
- DB 적재 커맨드: `python manage.py seed_engineering_jobs`
- Django 모델 변경: 없음
- DB 테이블 변경: 없음
- 생성 스크립트: 없음

## 주의사항

기존의 합성 데이터 생성 스크립트는 더 이상 사용하지 않는다. `scripts/generate_large_company_engineering_dataset.py` 는 의도적으로 중단 메시지만 출력한다.

## 서비스 영향

- 회사 조회, 기업 컬처/인재상 기반 분석에는 이 데이터셋을 그대로 사용할 수 있다.
- 직무 후보 목록과 직무별 추천 역량은 이 파일에서 더 이상 제공하지 않는다.
- `Job` 데이터가 필요하면 별도의 실제 채용공고 수집/검수 파이프라인으로 관리해야 한다.

## 추후 개선

- 회사별 출처 URL과 최종 검수일을 별도 관리
- 지원 기업 범위를 단계적으로 확대
- 실제 채용공고 원문 기반 `Job` 적재 경로 분리
- 관리자 검수 프로세스와 최신화 주기 추가
