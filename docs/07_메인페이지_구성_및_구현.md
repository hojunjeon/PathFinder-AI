# 메인페이지 구성 및 구현 정리

## 1. 작업 목적

PathFinder AI의 진입점을 메인 페이지로 통일하고, 사용자가 자연스럽게 다음 행동으로 이동하도록 흐름을 정리했다.

이번 수정의 핵심 목표는 다음과 같다.

1. 주소(`/`)로 처음 접속했을 때 메인 페이지가 보이도록 한다.
2. 로그인 전에는 로그인 버튼이 바로 보이도록 한다.
3. 로그인 후에도 처음 보여지는 화면을 로드맵 생성 페이지가 아니라 메인 페이지로 유지한다.
4. 메인 페이지는 기능을 나열하기보다, 간단한 설명과 버튼 중심의 흐름으로 구성한다.

---

## 2. 실제 변경 사항

### 2.1 메인 페이지 구조 단순화

**파일:** `frontend/src/views/HomeView.vue`

기존에는 메인 페이지에 여러 섹션과 많은 설명이 포함되어 있었다. 이번 수정에서는 이를 줄이고 아래 3개 영역 중심으로 재구성했다.

- 서비스 한 줄 소개와 핵심 메시지
- 버튼 중심의 주요 행동 영역
- 3단계 사용 흐름 안내
- 로그인 상태에 따른 다음 행동 안내

#### 로그인 전 화면

로그인하지 않은 사용자는 다음 버튼을 볼 수 있다.

- `로그인하고 시작하기`
- `채용시장 둘러보기`

#### 로그인 후 화면

로그인한 사용자는 다음 버튼을 볼 수 있다.

- `로드맵 생성하기`
- `내 프로필 정리하기`
- `이전 로드맵 보기`

### 2.2 사용자 흐름 정리

메인 페이지에 간단한 사용 흐름을 추가했다.

1. 내 정보 정리
2. 지원 공고 입력
3. 로드맵 확인

이 흐름은 사용자가 처음 서비스를 접했을 때 어떤 순서로 이용하면 되는지 바로 이해하도록 돕는다.

### 2.3 로그인 성공 후 이동 경로 변경

**파일:** `frontend/src/views/LoginView.vue`

기존에는 로그인 또는 회원가입 후 `/analyze/new`로 이동했다. 이를 `/`로 변경해, 로그인 후에도 메인 페이지가 먼저 보이도록 바꿨다.

이렇게 변경한 이유는 다음과 같다.

- 로그인 직후 바로 생성 화면으로 보내기보다, 메인 페이지에서 다음 행동을 선택하게 만들기 위해
- 사용자의 현재 상태에 맞는 버튼을 메인 페이지에서 안내하기 위해

### 2.4 상단 내비게이션 단순화

**파일:** `frontend/src/App.vue`

전역 내비게이션은 유지하되, 기능 접근은 메인 페이지 버튼 중심으로 유도하도록 단순화했다.

변경된 내용:

- 항상 보이는 메뉴를 최소화
- `홈` 이동 링크 유지
- 로그인 전에는 `로그인` 버튼 노출
- 로그인 후에는 `로그아웃` 버튼 노출
- 테마 토글은 유지
- 로그아웃 후 이동 경로를 `/login`이 아니라 `/`로 변경

이로써 상단 메뉴는 보조 역할만 하고, 실제 기능 진입은 메인 페이지에서 자연스럽게 이어지도록 정리했다.

## 3. 적용된 사용자 흐름

### 3.1 첫 방문 사용자

1. 주소창에 접속하면 메인 페이지가 열린다.
2. 서비스 설명을 읽는다.
3. `로그인하고 시작하기` 버튼으로 이동한다.
4. 로그인 후 다시 메인 페이지로 돌아온 뒤 필요한 기능을 선택한다.

### 3.2 로그인한 사용자

1. 메인 페이지가 먼저 열린다.
2. `로드맵 생성하기` 또는 `내 프로필 정리하기` 버튼으로 바로 이동한다.
3. 필요하면 `이전 로드맵 보기`로 이어서 확인한다.

## 4. 구현 상세

### 4.1 메인 페이지 구현 방식

- Vue `computed`를 사용해 로그인 여부에 따라 버튼과 안내 문구를 분기했다.
- 메인 페이지는 카드형 섹션 3개만 두고, 불필요한 정보는 줄였다.
- 서비스 설명은 짧고 명확하게 유지했다.
- 메인 CTA는 모두 버튼 형태로 배치했다.

### 4.2 로그인 상태 연동

- `useAuthStore()`로 로그인 상태를 읽는다.
- 로그인 여부에 따라 메인 페이지의 버튼 구성이 달라진다.
- 로그인 성공 후 리다이렉트도 `/`로 통일했다.

### 4.3 전역 내비게이션 정리

- `App.vue`에서 상단 링크를 최소화했다.
- 홈 진입과 로그인/로그아웃만 빠르게 처리하도록 구성했다.
- 기능별 이동은 메인 페이지에서 하도록 자연스럽게 유도했다.

## 7. 정리

이번 수정으로 PathFinder AI의 진입점이 더 명확해졌다.

- 첫 방문자는 메인 페이지에서 서비스 의미를 빠르게 이해한다.
- 로그인 전에는 로그인 버튼이 바로 보여 진입이 쉽다.
- 로그인 후에는 곧바로 로드맵 생성 화면으로 강제 이동하지 않고, 메인 페이지에서 필요한 기능을 선택한다.
- 메인 페이지는 과도한 설명보다 사용 흐름과 버튼 중심으로 정리되어 있다.
