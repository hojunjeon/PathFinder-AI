# 메인페이지 구성 및 구현 정리

## 1. 작업 목적

기존 프론트엔드의 루트 경로(/)는 곧바로 /analyze/new로 리다이렉트되어, 사용자가 서비스의 가치와 활용 흐름을 이해할 수 있는 진입 페이지가 없었다.

이번 작업의 목표는 다음과 같다.

1. 신규 방문자에게 PathFinder AI가 어떤 문제를 해결하는 서비스인지 설명한다.
2. 로그인 사용자에게는 바로 다음 행동(로드맵 생성, 히스토리 확인, 프로필 보강)으로 이동할 수 있는 홈 대시보드를 제공한다.
3. 서비스의 핵심 기능인 채용공고 분석, 자기소개서 연결, 면접 유형 기반 로드맵 생성, 채용시장 데이터 확인을 한 페이지에서 연결한다.
4. 구현 과정과 의사결정을 문서화해 이후 기획/디자인/개발 기준으로 재사용한다.

## 2. 현재 상태 분석

### 2.1 기존 라우팅

파일: frontend/src/router/index.js

기존 상태:

    { path: '/', redirect: '/analyze/new' }

문제점:

- 비로그인 사용자는 루트 진입 후 인증 가드에 의해 /login으로 이동해 서비스 맥락을 알기 어렵다.
- 로그인 사용자도 곧바로 분석 생성 폼으로 이동하므로, 히스토리/프로필/대시보드 같은 보조 기능을 발견하기 어렵다.
- 브랜드 홈 역할을 하는 페이지가 없어 전역 내비게이션의 PathFinder AI 로고 링크가 정보 구조상 약하다.

### 2.2 활용 가능한 기존 기능

| 기능 | 기존 경로 | 메인페이지에서의 역할 |
|---|---|---|
| 로그인/회원가입 | /login | 비로그인 사용자의 시작 CTA |
| 로드맵 생성 | /analyze/new | 로그인 사용자의 핵심 CTA |
| 분석 히스토리 | /history | 기존 결과 재방문 및 준비 상태 추적 |
| 프로필 | /profile | 분석 품질을 높이기 위한 개인 데이터 보강 |
| 채용시장 분석 | /dashboard | 서비스 신뢰도와 데이터 기반 관점 제시 |

## 3. 메인페이지에 포함한 요소

### 3.1 히어로 영역

목적:

- 서비스의 핵심 가치를 첫 화면에서 설명한다.
- 사용자가 가장 중요한 행동으로 바로 이동할 수 있게 한다.

구성:

- 서비스 가치 문구: “채용공고부터 면접 준비까지, 오늘 할 일을 한 번에 정리하세요.”
- 주요 CTA:
  - 로그인 상태: 새 로드맵 만들기, 프로필 보강하기
  - 비로그인 상태: 무료로 시작하기, 채용시장 먼저 보기
- 핵심 지표 카드:
  - 3단계 분석
  - 10K+ 채용시장 대시보드 데이터
  - 주차별 실행 로드맵
- 로드맵 결과 미리보기 UI:
  - 역량 매칭 예시
  - 주차별 계획 예시

### 3.2 빠른 시작 카드

목적:

- 사용자가 현재 상태에 맞는 다음 행동을 선택하게 한다.

로그인 사용자 카드:

1. 새 로드맵 생성 → /analyze/new
2. 분석 히스토리 확인 → /history
3. 프로필 보강 → /profile

비로그인 사용자 카드:

1. 서비스 시작 → /login
2. 채용시장 탐색 → /dashboard
3. 활용 흐름 이해 → 페이지 내 흐름 섹션

### 3.3 서비스 활용 흐름

목적:

- 사용자가 분석 생성 과정을 시작하기 전에 전체 단계를 이해하게 한다.

단계:

1. 채용공고 입력
2. 자기소개서 연결
3. 면접 유형 선택
4. 로드맵 실행

### 3.4 주요 기능 설명

목적:

- 단순한 마케팅 문구가 아니라 실제 구현된 기능과 연결해 서비스 신뢰도를 높인다.

포함 항목:

- 맥락 있는 로드맵
- 프로필 자산화
- 시장 데이터 참고
- 히스토리 관리

### 3.5 로그인 사용자 최근 로드맵

목적:

- 로그인 사용자가 홈에서 최근 작업을 바로 이어갈 수 있게 한다.

구현 방식:

- 로그인 상태에서만 GET /api/analyze/history/ 호출
- 최신 3개만 표시
- 결과가 없거나 API 연결 실패 시 빈 상태 UI 제공
- 각 항목 클릭 시 /analyze/:id로 이동

## 4. 구현 내용

### 4.1 새 Vue 페이지 추가

파일:

- frontend/src/views/HomeView.vue

주요 구현:

- Composition API 사용
- useAuthStore()로 로그인 상태 판별
- computed()로 로그인/비로그인별 빠른 시작 카드 분기
- onMounted()에서 로그인 사용자만 히스토리 API 호출
- API 실패 시 페이지 전체가 깨지지 않도록 catch에서 빈 목록 처리
- scoped CSS로 메인페이지 전용 레이아웃 구현
- 데스크톱/태블릿/모바일 반응형 그리드 구성

### 4.2 라우터 변경

파일:

- frontend/src/router/index.js

변경 전:

    { path: '/', redirect: '/analyze/new' }

변경 후:

    { path: '/', component: () => import('../views/HomeView.vue'), meta: { public: true } }

추가 변경:

    { path: '/dashboard', component: () => import('../views/DashboardView.vue'), meta: { public: true } }

/dashboard는 정적 jobs_careers.jsonl 기반 데이터를 사용하므로 비로그인 사용자도 서비스의 데이터 기반 관점을 미리 확인할 수 있도록 공개 경로로 조정했다.

### 4.3 전역 내비게이션 변경

파일:

- frontend/src/App.vue

변경 내용:

- 로그인 사용자 메뉴에 홈 링크 추가
- 비로그인 사용자 메뉴에 서비스 소개, 로그인 링크 추가
- 기존 로고 링크(/)가 실제 메인페이지로 동작하도록 정보 구조 정리

## 5. 사용자 관점의 기대 효과

| 사용자 상황 | 기존 경험 | 개선 후 경험 |
|---|---|---|
| 처음 방문 | 바로 로그인 또는 분석 폼으로 이동 | 서비스 가치와 흐름을 먼저 이해 |
| 로그인 후 재방문 | 분석 생성 화면으로 강제 이동 | 새 로드맵/히스토리/프로필 중 선택 |
| 분석 결과가 있는 사용자 | 히스토리 메뉴를 직접 찾아야 함 | 홈에서 최근 로드맵 3개 확인 |
| 공고는 없지만 탐색 중인 사용자 | 무엇부터 해야 할지 불명확 | 채용시장 대시보드로 먼저 탐색 |

## 6. 검증 결과

프론트엔드 기준으로 다음 검증을 수행했다.

    cd frontend
    npm.cmd run build
    npm.cmd test

결과:

- npm.cmd run build: 통과
- npm.cmd test: frontend design verification passed

참고:

- PowerShell 실행 정책으로 npm run build는 npm.ps1 실행이 차단되어 실패했다.
- 동일 명령을 Windows 실행 파일인 npm.cmd로 수행해 정상 검증했다.

## 8. 이후 개선 가능성

1. 최근 로드맵 카드에 역량 격차 요약이나 다음 학습 항목을 함께 표시한다.
