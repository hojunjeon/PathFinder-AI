# 분석 결과 페이지 가독성 및 준비 항목 개선

## 배경

분석 결과 페이지는 기존에 다음 정보를 한 화면에서 보여주고 있었다.

- 회사명과 직무명
- 역량 분석
- 직무 역량 매칭도
- 준비 항목 체크리스트
- 선택한 면접 유형과 생성일

하지만 실제 화면 사용성 관점에서 다음과 같은 개선 요청이 있었다.

1. 왼쪽 탭의 회사명 위에 표시되는 로고/이니셜은 불필요하므로 제거
2. 실제 페이지 구성 순서와 왼쪽 섹션 이동 순서가 동일하게 보이도록 정리
3. 왼쪽 탭의 `준비 항목` 옆 숫자 제거
4. `역량 분석` 영역을 단순 문장 목록이 아니라 키워드 중심으로 가독성 있게 정리
5. `준비 항목`에서는 학습 기준이나 근거를 제거하고, 개인에게 할 수 있는 질문과 답변 방향만 보여주도록 개선

## 변경 목표

| 목표 | 설명 |
|---|---|

| 화면 순서 일치 | 실제 본문 순서와 왼쪽 탭 순서를 `분석 요약 -> 역량 분석 -> 직무 매칭도 -> 준비 항목`으로 통일 |

| 역량 분석 가독성 개선 | 역량 문장을 키워드 제목과 보조 설명 형태로 표시 |
| 준비 항목 구조 개선 | 작은 카테고리/근거/학습 기준 중심 표시를 개인 맞춤 질문/답변 팁 중심으로 변경 |

## 상세 변경 내용

### 1. 왼쪽 회사 카드 로고 제거

기존에는 회사명 위에 회사명의 첫 글자를 표시하는 이니셜 박스가 있었다.

```vue
<div class="company-initial">{{ analysis.company_name?.charAt(0) || 'K' }}</div>
```

이 요소와 관련 CSS `.company-initial`을 제거했다.

변경 후 왼쪽 카드에는 다음 정보만 표시된다.

- 회사명
- 직무명

### 2. 페이지 구성 순서와 왼쪽 탭 순서 통일

기존 사이드바 순서는 실제 본문 순서와 다르게 보일 수 있었다.

변경 후 섹션 목록을 `pageSections` 배열로 관리한다.

```js
const pageSections = [
  { id: 'summary', label: '분석 요약' },
  { id: 'gap', label: '역량 분석' },
  { id: 'scores', label: '직무 매칭도' },
  { id: 'roadmap', label: '준비 항목' },
]
```

본문도 같은 anchor 순서를 사용한다.

```text
summary: 분석 요약
-> gap: 역량 분석
-> scores: 직무 매칭도
-> roadmap: 준비 항목
```

사이드바에는 `01`, `02`, `03`, `04` 순번을 함께 표시해 페이지 흐름을 더 쉽게 파악할 수 있도록 했다.

### 3. `준비 항목` 숫자 배지 제거

기존에는 왼쪽 탭의 `준비 항목` 옆에 `roadmapItems.length` 숫자가 표시됐다.

```vue
<span class="count" v-if="roadmapItems.length">{{ roadmapItems.length }}</span>
```

요청에 따라 해당 숫자 배지와 `.count` 스타일을 제거했다.

### 4. 역량 분석 키워드 카드화

기존 `역량 분석`은 다음과 같은 단순 목록이었다.

```text
- 프로젝트 경험
- 시스템 설계
- Python
```

변경 후에는 세 그룹으로 나누어 표시한다.

- 강점 키워드
- 보완 키워드
- 요구 역량 키워드

각 항목은 다음처럼 렌더링된다.

```text
키워드
보조 설명
```

문자열에 괄호, 콜론, 하이픈 등이 포함된 경우 가능한 범위에서 키워드와 설명을 나눈다.

예시:

```text
시스템 설계 - 대규모 트래픽 구조 설명 보완 필요
```

표시 결과:

```text
시스템 설계
대규모 트래픽 구조 설명 보완 필요
```

### 5. 준비 항목을 질문/답변 팁 중심으로 변경

기존 준비 항목은 작은 카테고리별로 다음 정보를 모두 보여줬다.

- 질문
- 답변 방향
- 근거
- 학습 기준
- 꼬리질문

요청에 따라 표시 정보를 단순화했다.

변경 후 표시 정보는 다음과 같다.

- 개인 맞춤 질문
- 답변 팁

`follow_up_questions`는 별도 카테고리로 분리하지 않고 개인 맞춤 질문 목록에 함께 포함했다.

`evidence`, `study_goal`, `sources`는 화면에서 표시하지 않는다.

### 6. LLM 프롬프트 방향 수정

신규 분석 결과도 같은 화면 구조에 맞도록 LLM 프롬프트를 수정했다.

핵심 변경 방향은 다음과 같다.

- 주차별 할 일 생성 금지
- 작은 카테고리 설명 중심이 아니라 개인 맞춤 질문 중심 생성
- 학습 기준, 세부 근거, 출처 나열은 출력하지 않도록 지시
- 답변 방향에 필요한 힌트만 간결히 작성하도록 지시

수정된 출력 의도는 다음과 같다.

```json
{
  "timeline_data": [
    {
      "category": "질문 묶음",
      "summary": "프로필·자기소개서·기업 정보·채용공고를 함께 반영한 질문 묶음 설명",
      "sources": [],
      "subtopics": [
        {
          "title": "질문 키워드",
          "done": false,
          "why": "",
          "question": "프로필·자기소개서·기업 정보·채용공고 기반 개인 맞춤 질문",
          "answer_guide": "답변 팁",
          "evidence": "",
          "study_goal": "",
          "follow_up_questions": ["꼬리질문"]
        }
      ]
    }
  ]
}
```

## 최종 상태

분석 결과 페이지는 다음 흐름으로 정리되었다.

```text
왼쪽 탭
01 분석 요약
02 역량 분석
03 직무 매칭도
04 준비 항목

본문
분석 요약
-> 역량 분석: 키워드 중심 카드
-> 직무 매칭도
-> 준비 항목: 개인 맞춤 질문 + 답변 팁
```

불필요하게 보이던 로고, 준비 항목 숫자, 근거, 학습 기준은 제거되었고, 역량 분석과 준비 항목은 면접 준비에 바로 활용하기 쉬운 형태로 정리되었다.
