# 프로필 이력사항 구조화 및 UI 개선

## 구현하고자 했던 기능

프로필 페이지를 구직 사이트의 이력서 입력 흐름처럼 항목별로 읽기 쉽고 체계적으로 작성할 수 있게 개선했다.

- 기본 정보, 경력사항, 프로젝트, 자격증, 수상내역을 단계별 카드 섹션으로 분리
- 항목 간 간격과 설명 문구를 추가해 가독성 개선
- 자기소개서 입력 섹션 제거
- 경력/프로젝트 기간을 `date` 입력으로 변경
- 자격증을 자격증명, 주관기관, 취득일, 등록번호로 분리
- 수상내역을 수상명, 주관기관, 수상일, 설명으로 분리
- 자격증과 수상내역을 줄바꿈 텍스트가 아니라 “항목 추가” 방식으로 여러 개 입력
- 기존 `period`, `date`, `org` 형태의 저장 데이터를 새 입력 구조로 읽을 수 있도록 호환 처리

## 수정한 파일

### `frontend/src/views/ProfileView.vue`

- 프로필 페이지 전체 레이아웃을 카드 기반 단계형 구조로 재구성했다.
- `CoverLetterForm` import와 자기소개서 섹션을 제거했다.
- 기존 줄바꿈 기반 `awardsText`, `certsText` 상태를 제거하고 `form.awards`, `form.certificates` 배열을 직접 편집하도록 변경했다.
- 저장 전 `pruneProfile()`에서 경력/프로젝트의 날짜 필드를 `period` 문자열로도 함께 만들어 기존 분석 로직과 호환되게 했다.
- 조회 시 `normalizeProfile()`로 기존 데이터와 새 데이터 구조를 같은 폼 구조로 변환한다.

### `frontend/src/components/profile/CareerForm.vue`

- 경력 항목을 회사명, 직무/직함, 고용 형태, 시작일, 종료일, 현재 재직 여부, 주요 업무 및 성과로 분리했다.
- 기간 입력은 텍스트가 아니라 `input type="date"`를 사용한다.
- “현재 재직 중”을 체크하면 종료일을 비운다.

### `frontend/src/components/profile/ProjectForm.vue`

- 프로젝트 항목을 프로젝트명, 역할, 시작일, 종료일, 기술 스택, 설명, 결과/성과로 분리했다.
- 프로젝트 기간도 `date` 입력을 사용한다.

### `frontend/src/components/profile/CertificateForm.vue`

- 새 컴포넌트를 추가했다.
- 자격증을 자격증명, 주관기관, 취득일, 등록번호로 구조화했다.
- `+ 자격증 추가` 버튼으로 여러 자격증을 배열 형태로 관리한다.

### `frontend/src/components/profile/AwardForm.vue`

- 새 컴포넌트를 추가했다.
- 수상내역을 수상명, 주관기관, 수상일, 설명으로 구조화했다.
- `+ 수상내역 추가` 버튼으로 여러 수상내역을 배열 형태로 관리한다.

### `frontend/tests/e2e/profile.spec.js`

- 자기소개서 섹션이 프로필 페이지에 표시되지 않는지 확인하는 테스트를 추가했다.
- 자격증/수상내역이 구조화된 JSON 배열로 저장되는지 확인하는 테스트를 추가했다.

## 동작 원리

백엔드의 `Profile` 모델은 `careers`, `projects`, `awards`, `certificates`를 `JSONField`로 저장한다. 따라서 DB 스키마를 바꾸지 않고도 프런트엔드에서 더 세분화된 객체 구조를 저장할 수 있다.

예를 들어 자격증은 다음과 같은 배열로 저장된다.

```json
[
  {
    "name": "정보처리기사",
    "issuer": "한국산업인력공단",
    "acquired_date": "2025-06-01",
    "credential_id": ""
  }
]
```

수상내역은 다음과 같은 배열로 저장된다.

```json
[
  {
    "title": "SSAFY 프로젝트 우수상",
    "issuer": "삼성청년SW아카데미",
    "award_date": "2025-12-15",
    "description": "팀 프로젝트에서 API 설계와 배포를 담당했습니다."
  }
]
```

기존 데이터 호환을 위해 `normalizeAward()`와 `normalizeCertificate()`는 예전 필드명도 함께 읽는다.

- 예전 수상 필드 `org`, `date` → 새 필드 `issuer`, `award_date`
- 예전 자격증 필드 `date` → 새 필드 `acquired_date`

경력과 프로젝트는 저장 시 날짜 필드와 함께 `period` 문자열도 만든다.

```js
period: `${start_date} ~ ${end_date}`
```

이렇게 하면 새 UI에서는 날짜 입력을 사용할 수 있고, 기존 분석 로직이 `period`를 참조해도 계속 동작한다.
