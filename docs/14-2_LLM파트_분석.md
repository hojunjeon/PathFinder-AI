# PathFinder AI: LLM 파트 2차 고도화 설계서

본 문서는 사용자의 실제 서비스 이용 목적("내 경험과 채용공고 내용을 매칭하여 공부해야 할 내용을 정리하고, 면접 답변을 보완하는 것")에 완벽히 부합하도록 역량 분석 및 준비 항목 구조를 개편하는 LLM 파트 고도화 설계서입니다.

## 1. 개선 방향 및 원칙

### 1.1 불필요한 맥락 제거
- **면접 유형 고려 제외**: PT 면접, 인성 면접 등 '유형'에 얽매여 질문이 왜곡되는 현상을 방지하고, 철저히 **직무 지식과 사용자 경험 검증**에 집중합니다.
- **불필요한 부연 설명 제거**: 역량 분석 카드 내의 장황한 답변 가이드나 분류 설명을 제거하고, 직관적인 키워드 위주로 재편합니다.

### 1.2 핵심 가치(Core Value) 재정의: '정리할 역량'
- 경험은 있지만 자소서에 설명이 부족하거나 직무 언어로 정제되지 않은 영역인 **'정리할 역량'**을 서비스의 핵심 기능으로 삼아, 사용자가 본인의 경험을 직무 역량으로 포장할 수 있도록 돕습니다.

## 2. 데이터 처리 파이프라인 설계

입력 데이터가 방대해지고 분석의 정확도가 요구되므로, LLM의 토큰 한계와 환각(Hallucination)을 막기 위해 **단일 프롬프트에서 '2-Step 파이프라인'으로 분리**하여 체계화합니다.

### Step 1: [진단] 경험-공고 매칭 및 역량 분류
1. **입력**: 채용공고 원문 + 사용자 프로필/자소서/이력서
2. **LLM 수행 목표**:
   - 채용공고를 분석하여 직무에 필요한 핵심 요구 역량 키워드를 추출.
   - 사용자의 경험과 대조하여 강점, 정리할 역량, 약점으로 3분할.
3. **출력물**: `competency_analysis` (역량 분석) 데이터

### Step 2: [생성] 세부 지식 구조화 및 면접 질문 도출
1. **입력**: 채용공고 원문 + 사용자 데이터 + Step 1의 분석 결과
2. **LLM 수행 목표**:
   - 도출된 직무 역량별로 하위 '직무 지식'을 전개.
   - 지식과 사용자 경험을 연결하고, 핵심 개념을 포함한 '맞춤형 면접 질문' 도출.
   - 해당 질문에 대한 어필 가이드(답변 관점) 작성.
3. **출력물**: `preparation_roadmap` (준비 항목) 데이터

## 3. 핵심 도메인 구조 및 UI/데이터 매칭

### 3.1 역량 분석 (Competency Analysis)
복잡한 설명을 제거하고, **핵심 키워드 기반의 3가지 상태**로만 나타내어 "무엇을 보완하고 공부해야 할지" 한눈에 파악할 수 있도록 합니다.

- **강점 역량 (Strength)**
  - 정의: 직무에 요구되며, 사용자의 경험(자소서/이력서)으로 잘 서술되어 있어 실제 경쟁력이 되는 역량.
- **정리할 역량 (Organize) ⭐ (핵심 기능)**
  - 정의: 직무에 필요한 경험을 해본 적은 있는 것 같으나, 서술이 부족하거나 핵심 기술 관점(직무 언어)으로 정리가 안 된 역량.
- **약점 역량 (Weakness)**
  - 정의: 직무 설명에는 명시되어 있으나, 사용자의 경험/이력에서는 전혀 찾을 수 없어 백지상태에서 공부해야 하는 역량.

### 3.2 준비 항목 (Preparation Roadmap)
채용공고 내용을 기반으로 도출된 최상위 키워드부터 실제 면접 질문까지 트리(Tree) 구조로 체계화합니다.

* **[상위 1 Depth] 직무 역량 키워드**: 채용공고 분석을 통해 도출된 핵심 업무 단위 (예: "대용량 트래픽 처리")
  * **[하위 2 Depth] 직무 지식 키워드**: 해당 역량을 발휘하기 위해 필요한 구체적 지식 단위 (예: "Redis 캐싱 전략")
    * **세부 구성 항목**:
      1. **업무 연결성**: 이 지식이 실제 공고의 업무에서 어떻게/왜 쓰이는지.
      2. **경험 매칭**: 나의 어떤 경험(프로젝트 등)과 직관적으로 연결되는지.
      3. **핵심 개념**: 공부해야 할 필수 개념 키워드.
      4. **예상 면접 질문**: 채용공고 요구사항과 나의 자소서/이력을 복합 고려하여, '핵심 개념'을 묻는 형태의 면접 질문.
      5. **어필 가이드 (답변 관점)**: 질문 의도를 뚫고, 내 경험을 직무/업무와 어떻게 연결 지어 대답할지 제시하는 팁.

## 4. LLM JSON 응답 스키마 (설계안)

LLM 서버가 최종적으로 백엔드로 반환해야 할 구조화된 JSON 포맷입니다.

```json
{
  "competency_analysis": {
    "strengths": ["MSA 설계", "RESTful API 개발"],
    "organize": ["데이터베이스 성능 최적화", "CI/CD 파이프라인"],
    "weaknesses": ["대용량 트래픽 분산 처리"]
  },
  "preparation_roadmap": [
    {
      "job_competency": "데이터베이스 성능 최적화",
      "sub_knowledges": [
        {
          "knowledge_keyword": "인덱스(Index) 설계 및 실행 계획",
          "task_connection": "서비스 오픈 시 대량 데이터 조회의 병목 현상을 방지하기 위해 필수적임.",
          "experience_match": "쇼핑몰 프로젝트의 '상품 검색 속도 30% 개선' 경험과 직접 연결",
          "core_concept": "B-Tree 구조, 커버링 인덱스, 쿼리 실행 계획(Explain)",
          "interview_question": "쇼핑몰 프로젝트에서 검색 속도를 30% 개선하셨다고 했는데, 당시 생성한 인덱스의 구조는 무엇이며 쿼리 실행 계획에서 어떤 부분을 중점적으로 확인하셨나요?",
          "appeal_perspective": "[경험 연결] 단순 인덱스 생성을 넘어, 실제 Explain 명령어로 병목을 찾았던 경험을 강조. [업무 연결] 이러한 분석력이 입사 후 쿼리 튜닝 업무에 어떻게 기여할 수 있는지 어필."

      ]
    }
  ]
```

## 5. 기대 효과 및 개선 포인트 반영 결과

1. **직관성 극대화**: 역량 분석 카드에서 구구절절한 설명을 빼고 키워드 단위의 3분할(강점/약점/정리)로 시각화하여 사용자의 인지 부하를 줄였습니다.
2. **'정리할 역량'의 발견**: "해본 적은 있지만 정리가 안 된" 경험을 AI가 찾아내어 직무 언어로 재포장하도록 유도함으로써, 서비스의 핵심 차별화 포인트가 됩니다.
3. **구조적인 면접 준비**: [공고 역량 → 필요 지식 → 내 경험 → 면접 질문 → 방어 논리]로 이어지는 논리적 흐름을 통해, 단순히 지식을 외우는 것이 아니라 **내 경험과 결합한 실전 답변**을 준비할 수 있게 되었습니다.
