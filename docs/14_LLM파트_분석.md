# PathFinder AI: LLM 파트 구조 및 동작 원리 분석

본 문서는 `13_면접_예상질문_AI_결과_설계.md`를 바탕으로 PathFinder AI 프로젝트 내 LLM(대규모 언어 모델) 파트의 아키텍처, 데이터 흐름, 설계 원칙 및 기능적 특징을 구조화하여 정리한 문서입니다.

## 1. LLM 파트 아키텍처 (시스템 구조)

PathFinder AI의 LLM 파트는 **Django 백엔드**와 **SSAFY GMS OpenAI Gateway** 사이에서 프록시 역할을 하는 **FastAPI 기반의 독립된 마이크로서비스(`llm_server`)**로 구성되어 있습니다.

- **프레임워크**: FastAPI
- **모델**: `gpt-5-nano` (SSAFY GMS API 경유)
- **주요 엔드포인트**: `/llm/roadmap`
- **역할**: 
  - 백엔드로부터 사용자 및 공고 데이터를 받아 LLM 프롬프트 생성
  - GMS 호출 및 응답 수신
  - 응답 데이터(JSON)의 정규화 및 오류 보정
  - 가공된 정형 데이터를 Django 백엔드로 반환

## 2. 전체 데이터 처리 파이프라인

데이터가 입력되고 최종적으로 분석 결과가 화면에 표시되기까지의 흐름은 다음과 같습니다.

1. **데이터 수집 및 전처리 (Django)**
   - 채용공고, 직무 DB, 사용자 프로필, 자기소개서 등의 흩어진 데이터를 수집하여 하나의 정형화된 JSON 형태로 구성합니다.
2. **프롬프트 조립 (FastAPI)**
   - `roadmap_prompt.py`를 통해 입력 데이터를 기반으로 시스템 지시문(System Prompt)과 결합합니다.
   - 이때 임의의 환각(Hallucination)을 막고 반드시 입력 데이터(특히 채용공고와 자소서)를 근거로 결과를 생성하도록 강력한 제약을 둡니다.
3. **LLM 추론 (GMS API)**
   - 구성된 프롬프트를 `gpt-5-nano` 모델에 전송합니다.
   - 안전한 JSON 생성을 위해 `max_completion_tokens: 8000`, `reasoning_effort: minimal` 옵션을 사용하여 추론 토큰 소진으로 인한 응답 잘림 문제를 방지합니다.
4. **응답 정규화 및 후처리 (FastAPI)**
   - LLM이 반환한 JSON이 완벽하지 않을 수 있으므로 `main.py`에서 정규화를 수행합니다.
   - 필드 누락 방어, 데이터 타입 보정(ex: 객체가 아닌 문자열 반환 시 객체화), 빈 값 제거 등 하위 호환성을 보장하는 과정을 거칩니다.
5. **저장 및 시각화 (Django & Vue)**
   - 검증된 데이터는 Django의 `JSONField`에 저장되며, Vue 프론트엔드로 전달되어 사용자에게 노출됩니다.

## 3. 핵심 설계 원칙 및 특징

단순히 AI에게 질문을 맡기는 것이 아니라, **결과의 신뢰성**과 **실제적인 면접 준비의 효용성**을 높이기 위한 엄격한 설계 원칙이 적용되어 있습니다.

### 3.1. 근거 없는 점수 제거 및 상태 기반 분석
AI가 임의로 생성하는 근거 없는 백분율 점수(ex. "직무 적합도 83%")를 완전히 제거했습니다. 대신 사용자의 경험과 직무 요구사항의 매칭도를 **4가지 명확한 상태**로 분류하여 제공합니다.
- **어필 가능 (`strength`)**: 직무 요구사항과 직접 연결되는 충분한 경험이 있음
- **답변 정리 (`articulate`)**: 유사 경험은 있으나, 직무 언어로의 재구성과 설명이 필요함
- **학습 필요 (`study`)**: 직무에 필요하지만 관련 경험/지식이 없어 새롭게 공부해야 함
- **판단 보류 (`insufficient_data`)**: 현재 입력된 정보만으로는 AI가 판단할 수 없음

### 3.2. 채용공고 최우선 (근거주의)
면접 질문이나 학습 로드맵을 생성할 때, 범용적인 직무 지식이 아니라 **사용자가 입력한 채용공고의 담당 업무와 요구 역량을 최우선 기준**으로 삼습니다. 모든 결과물(예상 질문, 보완 역량)에는 "왜 이 질문/학습이 필요한지"에 대한 명확한 출처(Evidence)가 매핑됩니다.

### 3.3. '모범 답안' 대신 '답변 가이드' 제공
완성된 모범 답안을 제공하는 대신, 면접관의 질문 의도, 답변의 구성 프레임워크(STAR 기법 등), 반드시 포함해야 할 내용, 피해야 할 답변 방식을 가이드로 제공하여 사용자가 본인의 경험을 스스로 구조화할 수 있게 돕습니다.

## 4. LLM 출력 데이터 구조 (결과물)

LLM의 출력 결과는 기능에 따라 크게 두 가지 핵심 구조로 나뉩니다.

1. **역량 분석 결과 (`competency_map`)**
   - 직무 키워드별로 사용자의 현재 준비 상태(Strength, Articulate, Study)를 빠르게 스캔할 수 있는 대시보드 형태의 데이터입니다. 긴 문장 대신 짧은 신호(Signal)와 즉각적인 다음 행동(Action)을 제시합니다.
2. **면접 준비 로드맵 및 예상 질문**
   - 채용공고의 '담당 업무'를 최상위 분류로 삼아, 필요한 직무 지식을 계층화합니다.
   - 각 지식별로 **개념(Concept)**, **경험(Experience)**, **적용(Application)**의 3가지 측면에서 예상 질문을 생성하여 실전 대비가 가능하도록 돕습니다.

## 5. 결론 및 요약

PathFinder AI의 LLM 파트는 단순한 텍스트 생성을 넘어, **데이터베이스처럼 엄격하게 구조화된 정보 추출 시스템**으로 기능합니다.
FastAPI 계층을 통한 강력한 데이터 전후처리, 근거 기반의 프롬프트 설계, 그리고 환각을 최소화하는 구조적 제약(정규화 계층)을 결합하여, 신뢰할 수 없는 AI의 한계를 극복하고 취업 준비생에게 실질적이고 구체적인 행동 가이드를 제공하는 것이 가장 큰 특징입니다.
