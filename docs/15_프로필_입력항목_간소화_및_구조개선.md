# 프로필 입력 항목 간소화 및 구조 개선

## 1. 변경 목적

프로필 정보가 LLM 채용 분석의 입력 자료로 사용되는 점을 유지하면서, 분석에 불필요한 입력 항목을 제거하고 사용자가 핵심 경험을 빠르게 작성할 수 있도록 개선했습니다.

## 2. 수정한 기능

### 경력 사항
- 회사명
- 직무
- 주요 업무 및 성과

고용 형태, 재직 기간, 재직 여부 입력은 제거했습니다.

### 프로젝트
- 프로젝트명
- 역할
- 기술 스택
- 프로젝트 설명
- 결과 및 성과

프로젝트 시작일과 종료일 입력은 제거했습니다.

### 자격증
- 자격증명

주관 기관, 취득일, 등록번호 입력은 제거했습니다.

### 수상내역
- 수상명
- 수상 설명

주관 기관과 수상일 입력은 제거했습니다.

## 3. 구현 방식

- 기존 `careers`, `projects`, `certificates`, `awards` JSON 컬렉션명은 유지했습니다.
- 화면에 표시되는 필드만 정규화하여 저장하도록 변경했습니다.
- 기존 데이터에 불필요한 필드가 포함되어 있어도 화면에서 제외하고 다음 저장 시 제거되도록 처리했습니다.
- 경력, 프로젝트, 자격증, 수상내역의 중복 폼 컴포넌트를 `ProfileEntryForm.vue` 하나로 통합했습니다.
- 입력 예시, 도움말, 추가·삭제 버튼과 접근성 라벨을 적용해 입력 편의성을 개선했습니다.
- 수상 설명이 LLM 프롬프트에 전달되도록 처리하고 기존 `org`, `issuer` 데이터도 호환되도록 유지했습니다.
