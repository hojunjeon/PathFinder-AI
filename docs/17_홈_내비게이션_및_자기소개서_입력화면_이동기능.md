# 홈 내비게이션 및 자기소개서 입력 화면 이동 기능

작성일: 2026-06-24  
작업 브랜치: `fix/everthing`

## 1. 작업 개요

이번 작업에서는 사용자가 주요 화면으로 더 쉽게 이동할 수 있도록 다음 두 가지 내비게이션 기능을 추가했다.

1. 로그인 후 표시되는 상단 메뉴에 `홈` 버튼 추가
2. 분석 결과 페이지의 왼쪽 사이드바에 `자기소개서 입력 화면` 버튼 추가

분석 결과를 생성하거나 계산하는 로직은 변경하지 않았다. 기존 Vue Router 경로와 분석 생성 화면을 재사용하여 화면 이동 기능만 구현했다.

---

## 2. 상단 메뉴 홈 버튼

### 구현 목적

기존 상단 메뉴에는 다음과 같은 기능 메뉴가 있었지만, 메인 화면으로 돌아가는 명시적인 `홈` 메뉴가 없었다.

- 로드맵 생성
- 히스토리
- 면접 후기
- 프로필
- 채용시장 분석

서비스 로고를 클릭해도 홈으로 이동할 수 있지만, 사용자가 기능을 바로 이해할 수 있도록 상단 메뉴에 별도의 `홈` 버튼을 추가했다.

### 수정 파일

- `frontend/src/App.vue`

### 구현 방식

로그인 사용자에게 표시되는 전역 내비게이션에 다음 Vue Router 링크를 추가했다.

```vue
<router-link to="/">홈</router-link>
```

`router-link`를 사용했기 때문에 전체 페이지를 새로 불러오지 않고 Vue Router가 `/` 경로의 `HomeView.vue`로 이동한다.

기존 전역 내비게이션 CSS를 그대로 사용하므로 별도의 색상이나 레이아웃 스타일은 추가하지 않았다. 기존 메뉴와 동일하게 다음 동작도 적용된다.

- 마우스를 올렸을 때 메뉴 글자 강조
- 현재 홈 경로에 있을 때 활성 메뉴 스타일 적용
- 모바일 화면에서 기존 메뉴와 함께 줄바꿈

## 3. 분석 결과 페이지의 자기소개서 입력 화면 버튼

### 구현 목적

분석 결과를 확인한 사용자가 자기소개서를 다시 입력하거나 새 분석을 시작하려면 기존에는 상단의 `로드맵 생성` 메뉴를 사용해야 했다.

결과 페이지 안에서도 입력 화면으로 바로 이동할 수 있도록 왼쪽 사이드바의 `섹션 이동` 영역 아래에 `자기소개서 입력 화면` 버튼을 추가했다.

### 수정 파일

- `frontend/src/views/AnalyzeResultView.vue`

### 구현 방식

분석 결과 페이지의 사이드바에 다음 링크를 추가했다.

```vue
<router-link class="cover-letter-link" to="/analyze/new">
  자기소개서 입력 화면
</router-link>
```

버튼을 누르면 기존 분석 생성 화면인 `/analyze/new`로 이동한다.

분석 생성 화면은 다음 순서로 구성되어 있으므로, 사용자는 채용공고 정보를 입력한 후 기존 자기소개서 입력 단계로 이동할 수 있다.

1. 채용공고 입력
2. 자기소개서 입력

이번 변경에서는 특정 분석 결과에 저장된 자기소개서 내용을 결과 화면에 표시하거나 자동으로 입력하지 않았다. 요청 범위에 맞게 기존 자기소개서 입력 화면으로 이동하는 기능만 구현했다.

### 버튼 스타일

기존 디자인 토큰과 사이드바 스타일을 재사용하여 다음과 같이 구성했다.

- 최소 높이 `44px`로 클릭 영역 확보
- 기존 `border`, `radius`, `surface` 색상 변수 사용
- 사이드바 너비에 맞춘 중앙 정렬
- hover 시 테두리와 글자를 accent 색상으로 강조
- 다크 모드에서도 기존 CSS 변수에 따라 자동 대응

새로운 색상이나 별도의 디자인 시스템은 추가하지 않았다.

---

## 4. 분석 기능에 미치는 영향

분석 관련 데이터 처리와 로드맵 생성 로직은 수정하지 않았다.

변경하지 않은 주요 영역은 다음과 같다.

- 분석 생성 API 호출
- LLM 요청 및 응답 처리
- 역량 분석 데이터
- 근거 커버리지 계산
- 로드맵 진행률 계산
- 분석 결과 저장 및 조회

이번 작업의 기능 변경 범위는 Vue Router를 통한 화면 이동과 링크 UI 추가로 제한했다.

## 5. 테스트 변경

### 수정 파일

- `frontend/tests/e2e/home.spec.js`
- `frontend/tests/e2e/analyze-flow.spec.js`

### 홈 메뉴 테스트

로그인한 사용자가 홈 화면에 접속했을 때 상단 `주요 메뉴` 내비게이션에 `홈` 링크가 표시되는지 확인하도록 테스트를 추가했다.

검증 항목:

- 로그인 상태에서 `홈` 링크가 표시됨
- 링크의 이름이 다른 홈 화면 CTA와 혼동되지 않도록 정확히 일치하는 메뉴를 조회함

### 자기소개서 입력 화면 이동 테스트

분석 결과가 렌더링된 후 왼쪽 사이드바의 `자기소개서 입력 화면` 링크를 클릭하는 흐름을 추가했다.

검증 항목:

1. 결과 페이지에 이동 링크가 표시됨
2. 링크 클릭 후 URL이 `/analyze/new`로 변경됨
3. 분석 생성 화면의 첫 단계 제목인 `어떤 회사에 지원했나요?`가 표시됨

### 기존 E2E 계약 정리

관련 E2E 테스트를 실행하는 과정에서 분석 생성 요청이 과거의 `job_id`를 기대하고 있는 것을 확인했다.

현재 프론트엔드는 다음 구조를 사용한다.

- `company_id`
- 구조화된 `job_posting`
- `job_posting_text`
- `submitted_cover_letter`

따라서 테스트의 요청 payload 기대값을 현재 구현과 일치하도록 수정했다. 실제 분석 코드는 변경하지 않았다.

## 8. 사용자 흐름

### 홈 이동

```text
로그인
→ 상단 메뉴의 홈 클릭
→ 메인 화면(/) 이동
```

### 분석 결과에서 자기소개서 입력 화면 이동

```text
분석 결과 화면(/analyze/:id)
→ 왼쪽 사이드바의 자기소개서 입력 화면 클릭
→ 로드맵 생성 화면(/analyze/new) 이동
→ 채용공고 입력
→ 자기소개서 입력
```
