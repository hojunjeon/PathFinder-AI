# 전역 내비게이션 로그인 계정 표시

## 1. 작업 개요

로그인 후 전역 내비게이션에서 현재 사용 중인 계정을 바로 확인할 수 있도록 계정 표시 영역을 추가했다.

파일명은 기존 문서의 마지막 순번인 `17` 다음 순번을 사용하고, 작업 핵심인 **전역 내비게이션의 로그인 계정 표시**를 요약해 `18_전역_내비게이션_로그인_계정_표시.md`로 정했다.

## 2. 구현한 기능

- 로그인 상태에서 원형 이니셜 아바타를 표시한다.
- 프로필 이름이 있으면 이름을 주 식별자로 표시한다.
- 이메일을 보조 식별자로 함께 표시한다.
- 이름이 없으면 이메일을 주 식별자로 사용한다.
- 계정 표시를 누르면 `/profile`로 이동한다.
- 모바일에서는 계정 정보를 메뉴 첫 줄에 넓게 배치한다.
- 로그아웃하면 저장된 계정 정보도 즉시 제거한다.
- 프로필에서 이름을 수정하면 전역 내비게이션에도 즉시 반영한다.

## 3. UI 결정

여러 웹 서비스에서 일반적으로 사용하는 **아바타 + 표시 이름 + 이메일** 계정 식별 패턴을 적용했다.

이 패턴은 이메일만 표시하는 방식보다 사람이 계정을 빠르게 구분하기 쉽고, 동일하거나 비슷한 이름을 사용하는 경우 이메일로 다시 확인할 수 있다는 장점이 있다. 별도 팝업 메뉴를 추가하지 않고 기존 프로필 링크 역할까지 통합해 내비게이션 복잡도는 최소화했다.

## 4. 구현 방식

### 백엔드

`backend/accounts/serializers.py`의 `ProfileSerializer`에 읽기 전용 `email` 필드를 추가했다. 기존 `/api/profile/` 요청 한 번으로 프로필 이름과 로그인 이메일을 함께 가져오며, 프로필 수정 요청으로 계정 이메일을 변경할 수 없다.

### 인증 상태 저장소

`frontend/src/stores/auth.js`에 다음 전역 상태와 동작을 추가했다.

- `currentUser`: 현재 계정의 이름과 이메일
- `currentUserLabel`: 이름 우선, 이름이 없으면 이메일을 반환하는 표시값
- `currentUserInitial`: 아바타에 사용할 첫 글자
- `fetchCurrentUser()`: 새로고침 후 `/api/profile/`에서 계정 복원
- `updateCurrentUser()`: 프로필 저장 후 내비게이션 상태 동기화

로그인 직후에는 입력한 이메일을 즉시 표시하고, 새로고침 시에는 인증된 프로필 API 응답을 기준으로 정확한 이름과 이메일을 복원한다.

### 전역 내비게이션

`frontend/src/App.vue`에 `/profile`로 이동하는 `account-indicator` 링크를 추가했다. 화면 낭독기가 목적을 알 수 있도록 `aria-label="현재 로그인 계정 프로필"`을 적용했다.

`frontend/src/style.css`에는 원형 이니셜 아바타, 이름·이메일 정보 계층, 긴 텍스트 말줄임, 데스크톱 구분선, 모바일 독립 행 배치를 추가했다.

### 프로필 동기화

`frontend/src/views/ProfileView.vue`에서 프로필 조회 및 저장 성공 시 `authStore.updateCurrentUser()`를 호출한다. 이름을 저장한 뒤 페이지를 새로고침하지 않아도 내비게이션 이름이 즉시 바뀐다.

## 5. 개인정보 및 보안 범위

- 이메일은 인증된 현재 사용자 본인의 정보만 반환한다.
- `/api/profile/`은 기존과 동일하게 인증이 필요한 API다.
- 이메일은 읽기 전용이므로 프로필 저장 API를 통한 계정 변경을 차단한다.
- 비밀번호, 토큰 등 민감 인증 정보는 UI에 표시하지 않는다.

## 6. 테스트

### 백엔드

- 프로필 조회 응답에 로그인 사용자의 이메일이 포함되는지 확인
- 프로필 수정 요청에 다른 이메일을 보내도 실제 계정 이메일이 변경되지 않는지 확인

### 프론트엔드 E2E

- 저장된 토큰으로 접속했을 때 이름과 이메일이 내비게이션에 표시되는지 확인
- 로그인 직후 입력한 이메일이 계정 표시 영역에 즉시 나타나는지 확인
