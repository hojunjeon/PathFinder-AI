## 1. 서비스 개요

### 1.1 목적

"서류 합격 후 면접을 어떻게 준비해야 할지 모르는 취업 준비생"의 문제를 해결한다.

### 1.2 핵심 가치

- 사용자 이력(프로필 + 자기소개서)과 채용공고 사이의 역량 Gap을 AI가 정량 분석
- 개인화된 면접 준비 로드맵(우선순위, 예상 질문, 학습 방향)을 자동 생성
- 채용 시장 데이터를 시각화하여 취업 전략 수립 지원

### 1.3 주요 사용자

| 사용자 유형 | 설명 |
|------------|------|
| 취업 준비생 | 서류 합격 후 면접 준비가 필요한 사용자 (주 타겟) |
| 이직 준비자 | 현직 직장인으로 경력직 면접 준비가 필요한 사용자 |
| 면접 경험 공유자 | 면접 후기를 커뮤니티에 공유하는 사용자 |

## 2. 기능 요구사항

### 2.1 인증 (FR-AUTH)

#### FR-AUTH-01: 회원가입

- **설명**: 이메일과 비밀번호로 신규 계정을 생성한다.
- **입력**: `email`, `password`, `terms_agreed_at`, `privacy_agreed_at`
- **처리**: `accounts.User` (AbstractBaseUser) 모델에 저장. 이메일 소문자 정규화 및 중복 검사.
- **출력**: 201 Created + JWT Access/Refresh Token
- **예외**: 이메일 중복 시 400 Bad Request

#### FR-AUTH-02: 로그인

- **설명**: 이메일과 비밀번호로 인증하여 JWT 토큰을 발급한다.
- **입력**: `email`, `password`
- **처리**: `AbstractBaseUser.check_password()` 검증
- **출력**: 200 OK + `access_token`, `refresh_token`
- **예외**: 인증 실패 시 401 Unauthorized

#### FR-AUTH-03: 토큰 갱신

- **설명**: Refresh Token으로 새 Access Token을 발급한다.
- **입력**: `refresh_token`
- **처리**: SimpleJWT `TokenRefreshView` 활용
- **출력**: 200 OK + 새 `access_token`
- **예외**: Refresh Token 만료 시 401 Unauthorized → 재로그인 유도

#### FR-AUTH-04: 로그아웃

- **설명**: 클라이언트 측 JWT 토큰을 삭제한다.
- **처리**: 프론트엔드에서 Pinia store의 토큰 초기화

#### FR-AUTH-05: 이용약관 및 개인정보처리방침 동의

- **설명**: 회원가입 시 이용약관 동의 시각(`terms_agreed_at`)과 개인정보처리방침 동의 시각(`privacy_agreed_at`)을 기록한다.
- **처리**: 동의 없이 가입 불가

### 2.2 프로필 관리 (FR-PROFILE)

#### FR-PROFILE-01: 프로필 조회

- **설명**: 로그인한 사용자의 프로필(이력 정보)을 조회한다.
- **출력**: `name`, `major`, `education`, `careers[]`, `projects[]`, `awards[]`, `certificates[]`

#### FR-PROFILE-02: 프로필 생성 및 수정

- **설명**: 사용자의 이력 정보를 입력하거나 수정한다. `Profile` 모델이 없으면 자동 생성.
- **입력**:
  - `name`: 이름
  - `major`: 전공
  - `education`: 학력 (예: "컴퓨터공학과 학사 졸업")
  - `careers[]`: 경력 목록 (`[{company, title, description}]`)
  - `projects[]`: 프로젝트 목록 (`[{name, role, stack, description, result}]`)
  - `awards[]`: 수상 목록 (`[{title, description}]`)
  - `certificates[]`: 자격증 목록 (`[{name}]`)
- **처리**: `PUT /profile/` — 전체 프로필 교체

#### FR-PROFILE-03: 반복 항목 관리

- **설명**: 경력, 프로젝트, 수상, 자격증을 동적으로 추가·삭제한다.
- **처리**: 프론트엔드 Vue 3 컴포넌트에서 배열 추가/삭제 후 전체 저장

#### FR-PROFILE-04: 프로필 AI 분석 연동

- **설명**: 저장된 프로필이 분석 생성 시 자동으로 LLM 컨텍스트에 포함된다.
- **처리**: `build_llm_payload()`에서 `user.profile` 자동 조회

#### FR-PROFILE-05: 프로필 없을 때 분석 가능

- **설명**: 프로필이 없어도 채용공고와 자기소개서만으로 분석 생성이 가능하다. 단, 결과 품질은 프로필 있을 때보다 낮을 수 있음을 안내한다.

### 2.3 AI 분석 (FR-ANALYZE)

#### FR-ANALYZE-01: 채용공고 URL 자동 해석

- **설명**: 채용공고 URL을 입력하면 지원 기업을 자동으로 식별하고 관련 직무 정보를 제안한다.
- **입력**: 채용공고 URL
- **처리**:
  1. `resolve_company_from_url()`: URL 호스트명과 별칭 테이블(`COMPANY_URL_ALIASES`) 매칭
  2. `fetch_job_posting_text()`: httpx + BeautifulSoup4로 HTML 스크래핑, 최대 8,000자 추출
  3. SSRF 방어: `is_safe_job_posting_url()`로 private IP·루프백 차단
- **출력**: 매칭된 기업 정보 + 직무 목록
- **예외**: SPA 렌더링 사이트는 텍스트 추출 불가 → 직접 입력 fallback 안내

#### FR-ANALYZE-02: 채용공고 직접 입력 (수동 fallback)

- **설명**: URL 스크래핑이 불가능한 경우 또는 URL 없이 채용공고 내용을 직접 텍스트로 입력할 수 있다.
- **입력**: `company_name`, `job_title`, `responsibilities`, `requirements`, `preferred_qualifications`

#### FR-ANALYZE-03: 자기소개서 입력

- **설명**: 면접 준비를 위해 지원 회사에 제출한 자기소개서를 Q&A 항목으로 입력한다.
- **입력**: `submitted_cover_letter_items[]` (`[{question, answer}]`)
- **처리**: `CoverLetter` 모델에 저장, Analysis와 연결

#### FR-ANALYZE-04: 면접 유형 선택

- **설명**: 어떤 면접 유형에 대비할지 선택한다.
- **선택지**: `InterviewType` 모델의 `code` 기반 (예: 기술면접, 임원면접, PT면접, 기타)
- **입력**: `selected_interview_types[]`, `interview_type_etc_text`

#### FR-ANALYZE-05: AI 로드맵 생성

- **설명**: 입력된 모든 정보를 종합하여 개인화된 면접 준비 로드맵을 생성한다.
- **처리**:
  1. `build_llm_payload()`: 6단계 파이프라인 (Step 1~4)
  2. `call_llm_server()`: FastAPI LLM 서버 POST /llm/roadmap
  3. 타임라인 2-pass 품질 보증
  4. `normalize_llm_result()`: 결과 정규화
  5. `Analysis` 모델에 저장 (`status: done`)
- **출력**: Analysis ID → 결과 화면으로 리다이렉트
- **처리 시간**: 일반적으로 20~60초 소요
- **예외**: LLM 서버 오류 시 `status: failed`, 503 반환

#### FR-ANALYZE-06: 분석 결과 조회

- **설명**: 생성된 분석 결과를 조회한다.
- **결과 구성**:
  - `competency_gap.competency_map[]`: 역량별 status/radar_score/job_score
  - `competency_gap.strengths[]`: 어필 가능 강점
  - `competency_gap.gaps[]`: 보완 필요 역량
  - `timeline_data[]`: 담당업무별 subtopics + 예상 질문 체크리스트
  - `text_roadmap`: 개인화 준비 요약 텍스트

#### FR-ANALYZE-07: 분석 히스토리 조회

- **설명**: 로그인한 사용자의 모든 분석 요청 이력을 최신순으로 조회한다.
- **출력**: Analysis 목록 (생성일, 회사명, 직무, 면접 유형, 상태)
- **처리**: `Analysis.objects.filter(user=request.user)` — 타인의 분석 접근 불가

### 2.4 분석 히스토리 (FR-HISTORY)

#### FR-HISTORY-01: 히스토리 목록 조회

- **설명**: 과거에 생성한 모든 분석 요청을 최신순으로 목록 조회한다.
- **출력**: 생성일, 회사명, 직무, 면접 유형, 상태(`pending/done/failed`)

#### FR-HISTORY-02: 히스토리 상세 열람

- **설명**: 과거 분석 결과를 다시 조회한다. (`/analyze/:id`로 이동)
- **처리**: `status: done`인 경우에만 상세 내용 표시

### 2.5 채용시장 대시보드 (FR-DASH)

#### FR-DASH-01: 산업별 연봉 vs 지원자 수 차트

- **설명**: 산업별 평균 연봉과 평균 지원자 수를 이중 축 혼합 차트로 표시한다.
- **데이터**: `jobs_careers.jsonl` 10,000건 기반 통계

#### FR-DASH-02: 직급별 지원자 분포 차트

- **설명**: 신입/대리/과장 등 직급별 채용 수요를 도넛 차트로 표시한다.

#### FR-DASH-03: 경력 요구조건 트렌드 차트

- **설명**: 채용 공고의 요구 경력 분포 추이를 라인 차트로 표시한다.

#### FR-DASH-04: 급여 분포도 차트

- **설명**: 전체 데이터셋의 연봉 분포를 영역 차트로 표시한다.

#### FR-DASH-05: 다중 필터 연동

- **설명**: 산업군 선택, 경력 범위 슬라이더, 기업 검색창 필터가 모든 차트에 동시 반영된다.
- **처리**: Vue 3 `watch()`로 필터 상태 감시 → 데이터 필터링 → 차트 업데이트

#### FR-DASH-06: 차트 PNG 다운로드

- **설명**: 각 차트를 PNG 파일로 저장한다.
- **처리**: `canvas.toDataURL('image/png')` + `<a>` 태그 다운로드

### 2.6 면접 후기 커뮤니티 (FR-COMM)

#### FR-COMM-01: 후기 목록 조회

- **설명**: 모든 사용자의 면접 후기를 최신순으로 조회한다.
- **출력**: `InterviewReview` 목록 (회사명, 직무, 면접 유형, 날짜, 난이도, 합격 여부)

#### FR-COMM-02: 후기 작성

- **설명**: 면접 경험을 기록하고 공유한다.
- **입력**:
  - `company_name`: 회사명 (필수)
  - `job_title`: 직무명 (필수)
  - `title`: 후기 제목 (최대 120자)
  - `interview_type`: 면접 유형 (선택)
  - `interview_date`: 면접 날짜 (선택)
  - `difficulty`: 난이도 1~5 (기본값: 3)
  - `result_status`: 합격 여부 (`passed/failed/pending/unknown`)
  - `interview_questions`: 면접 질문 (선택)
  - `content`: 후기 본문 (필수)
  - `tips`: 준비 팁 (선택)
- **처리**: `InterviewReview` 모델 생성

#### FR-COMM-03: 후기 상세 조회

- **설명**: 특정 후기의 전체 내용을 조회한다.
- **접근**: 비로그인 사용자도 조회 가능

#### FR-COMM-04: 후기 수정

- **설명**: 본인이 작성한 후기를 수정한다.
- **처리**: `PATCH /community/reviews/{id}/` — `user=request.user` 권한 검증

#### FR-COMM-05: 후기 삭제

- **설명**: 본인이 작성한 후기를 삭제한다.
- **처리**: `DELETE /community/reviews/{id}/` — `user=request.user` 권한 검증
- **예외**: 타인의 후기 삭제 시도 시 403 Forbidden

| 코드 | 요구사항 | 구현 방식 |

| NFR-SEC-05 | Prompt Injection 방어 | 사용자 입력을 `private-evidence` fenced block으로 격리 |
| NFR-SEC-06 | CORS 설정 | `django-cors-headers`로 허용 오리진 제한 |
| NFR-SEC-07 | 타인 데이터 접근 방지 | 모든 API에서 `user=request.user` 조건 필터링 |

### 3.3 가용성 (NFR-AVAIL)

| 코드 | 요구사항 | 구현 방식 |
|------|----------|-----------|
| NFR-AVAIL-01 | LLM 서버 장애 시 서비스 지속 | `GMS_KEY` 없으면 Mock 응답 (`roadmap_mock.py`) |

| NFR-AVAIL-04 | Analysis 상태 추적 | `pending / done / failed` 상태로 실패 분석 추적 가능 |

### 3.4 유지보수성 (NFR-MAINT)

| 코드 | 요구사항 | 구현 방식 |

| NFR-MAINT-05 | 문서화 | README.md + docs/ 폴더 설계 문서 유지 |

## 4. 제약사항

### CONST-01: 개발 환경 제약

- **런타임**: Python 3.11 이상, Node.js (npm 포함)
- **운영 체제**: Windows 환경 기준 (`run-dev.bat`)
- **데이터베이스**: 개발 환경은 SQLite. 운영 환경은 PostgreSQL 전환 예정

### CONST-02: LLM API 제약

- **API**: SSAFY GMS Gateway 경유 `gpt-5-nano` 전용
- **엔드포인트**: `https://gms.ssafy.io/gmsapi/api.openai.com/v1/chat/completions`
- **GMS_KEY 없을 때**: Mock 응답 자동 반환 (`roadmap_mock.py`)
- **최대 출력 토큰**: `GMS_MAX_COMPLETION_TOKENS=20000`
- **추론 노력**: `GMS_REASONING_EFFORT=minimal`

### CONST-03: 스크래핑 제약

- **정적 HTML만 지원**: SPA 렌더링 사이트(원티드, 쿠팡잡 등)는 스크래핑 불가
- **SSRF 방어**: 사설 IP 대역, 루프백, localhost URL은 스크래핑 거부
- **최대 텍스트 길이**: 8,000자로 절단

### CONST-04: 기업 지원 범위 제약

- **Knowledge Graph**: Fixture 기반. 실시간 크롤링 미지원 (MVP 이후 과제)
- **지원 기업 수**: `Company.roadmap_supported=True`인 기업만 Knowledge Graph 활용
- **URL 별칭 테이블**: `COMPANY_URL_ALIASES`에 등록된 도메인만 URL 자동 해석 지원

### CONST-05: 사용자 데이터 접근 제약

- 모든 분석(Analysis), 커버레터(CoverLetter), 면접 후기(InterviewReview) 데이터는 작성자 본인만 조회/수정/삭제 가능
- 커뮤니티 후기 목록 및 상세 조회는 비로그인 사용자도 가능 (공개 데이터)

## 5. 용어 정의

| 용어 | 정의 |
|------|------|
| **Analysis** | 사용자의 채용공고 + 자기소개서 + 프로필을 종합하여 LLM이 생성한 면접 준비 결과물. `competency_gap`, `timeline_data`, `text_roadmap`으로 구성 |
| **competency_gap** | LLM이 분석한 역량 Gap 결과. `competency_map` (역량별 status/score), `strengths` (강점), `gaps` (보완 역량), `study_priorities` (학습 우선순위) 포함 |
| **timeline_data** | 채용공고의 담당업무별 준비 항목. `category`(담당업무), `subtopics`(직무 지식), `questions`(예상 면접 질문) 구조 |
| **GraphRAG** | Knowledge Graph를 RAG(Retrieval-Augmented Generation) 컨텍스트로 활용하는 패턴. 기업 사실 정보를 subject-predicate-object 삼중 구조로 저장 후 검색하여 LLM에 주입 |
| **CompanyKnowledgeFact** | 승인된 공개 기업 정보. `subject`(주체), `predicate`(관계), `object`(내용)의 삼중 구조 |
| **InterviewType** | 면접 유형 마스터 데이터. `code`(기술/임원/PT 등), `label`(표시명), `description`(설명) |
| **strength** | 역량 Gap 분류 중 하나. 프로필/자소서에서 직접 경험이 확인되어 면접에서 바로 어필 가능한 역량 |
| **articulate** | 역량 Gap 분류 중 하나. 유사 경험이 있어 직무 언어로 전환하면 어필 가능한 역량 |
| **study** | 역량 Gap 분류 중 하나. 직무 요구 역량이지만 경험 근거가 없어 학습이 필요한 역량 |
| **insufficient_data** | 역량 Gap 분류 중 하나. 채용공고 또는 사용자 입력이 부족하여 판단이 불가능한 역량 |
| **2-pass 품질 보증** | LLM 1차 응답의 timeline_data에 누락된 담당업무가 있을 때, 수리 프롬프트로 2차 GPT 호출하여 병합하는 품질 향상 메커니즘 |
| **GMS Gateway** | SSAFY에서 제공하는 OpenAI 호환 API 게이트웨이. `gpt-5-nano` 모델 사용 |
| **X-Internal-Token** | Django 백엔드와 FastAPI LLM 서버 간의 내부 인증에 사용하는 비밀 토큰. 환경변수로 관리 |
| **JobPosting** | 사용자가 입력하거나 URL에서 추출한 채용공고 정보. `Company`와 연결될 수 있음 |
| **Profile** | 사용자의 이력 정보. `careers`, `projects`, `awards`, `certificates`를 JSONField로 저장 |
| **radar_score** | competency_map에서 프로필/자소서 기반으로 산정한 현재 역량 점수 (0~100) |
| **job_score** | competency_map에서 채용공고/직무 DB 기반으로 산정한 기업 요구 수준 (0~100) |
| **SSRF** | Server-Side Request Forgery. 서버가 공격자가 지정한 내부 주소에 요청을 보내도록 유도하는 공격. `is_safe_job_posting_url()`로 방어 |
